# Internship CRM

Внутренняя CRM-система для управления студентами, компаниями-партнёрами и стажировками карьерного центра.

## Проблема

Карьерный центр ведёт студентов и компании-партнёров в нескольких разрозненных Google-таблицах. Это приводит к:
- потере контекста между менеджерами (кто кому когда написал);
- просроченным контактам со студентами и компаниями;
- невозможности увидеть «общую картину» — кто сейчас на стажировке, у кого зависло резюме, какие компании на паузе;
- ручному дублированию данных между листами.

Система собирает всё в одном месте, подсвечивает то, что требует внимания, и снимает рутину по контролю сроков.

## Пользователь

Внутренняя команда карьерного центра — 2–5 человек:
- **Админ** — полный доступ ко всем данным и настройкам.
- **VIP-менеджер** — может читать карточки и менять статусы связок «студент ↔ компания», без удаления и системных правок.

Объём данных, под который сделана система: ~500–800 активных студентов, ~50–200 компаний.

## Основная функция

CRM ведёт жизненный цикл стажировки от появления студента до итогового статуса:
1. Заводит студента и его направления (Python, QA, Frontend, Data Analytics и др.).
2. Хранит карточки компаний-партнёров с их статусом найма и историей контактов.
3. Связывает студента с компанией через сущность Placement и трекает статус (отправлен → проходит собеседование → на стажировке → завершено).
4. Подсвечивает зависшие резюме, превышенные сроки стажировок и компании, давно стоящие на паузе.
5. Показывает менеджеру одной страницей: «что требует моего внимания сегодня».

## Формат результата

Пользователь работает в **веб-админке Django** (Django Admin + тема django-unfold, фиолетовая палитра, тёмная/светлая темы):

- **Главная** — кастомный дашборд: счётчики по статусам, алерты «зависло / просрочено / на паузе», список «сегодняшние контакты». Каждая карточка кликабельна и ведёт в отфильтрованный список.
- **Студенты** — таблица с круглыми аватарами-инициалами, цветная индикация по сроку следующего контакта (просрочено / сегодня / скоро), Telegram-индикатор, колонка «куда направлен / стажируется». Статусы меняются прямо из списка (inline-edit).
- **Компании** — список с фильтрами по статусу найма и направлениям. В карточке подсвечены «зависшие» Placement.
- **Стажировки (Placements)** — связки студент↔компания с историей статусов, сроком, плановой длительностью.
- **Комментарии и файлы** — прямо в карточках студента/компании (через GenericForeignKey).
- **Импорт** — management-команды для загрузки текущих Google-таблиц (студенты, компании) одной командой.

## Минимальный функционал (MVP — первая итерация)

Закрыто и работает:
- ✅ Модели: студенты, компании, связки (Placement), комментарии, пользователи и роли.
- ✅ Кастомизированная Django-админка с поиском, фильтрами, инлайнами, кастомными actions.
- ✅ Дашборд с алертами «что требует внимания».
- ✅ Группы прав: `admins` и `vip_managers`.
- ✅ Импорт студентов и компаний из CSV (Google Sheets export) через management-команды.
- ✅ Подсветка зависших Placement и компаний на паузе.
- ✅ Inline-редактирование статусов в списках.
- ✅ Тема django-unfold с фиолетовой палитрой, переключение dark/light.
- ✅ Docker Compose для локалки и прода (Postgres + Django + nginx).
- ✅ 98 unit/integration-тестов.

### Что НЕ в MVP — уходит во вторую итерацию

- ⏳ **Telegram-боты** (студенческий и менеджерский) — онбординг, привязка `chat_id`.
- ⏳ **Массовые рассылки** студентам через бот с предпросмотром и обработкой блокировок.
- ⏳ **Автопуши менеджерам** в групповой чат (зависшие резюме, превышенные сроки, паузы компаний).
- ⏳ **Опросники студентам** через inline-кнопки бота.
- ⏳ **Утренний дайджест** менеджерам.
- ⏳ **Аналитика и conversion-метрики** (среднее время от отправки до решения, воронка).

## Технологии

- **Backend:** Python 3.12, Django 5, Django REST Framework
- **БД:** PostgreSQL 16 (в Docker), для быстрого старта — SQLite
- **Админка:** Django Admin + django-unfold (тёмная тема, фиолетовая палитра)
- **Фоновые задачи:** django-q2 (без Redis — очередь живёт в Postgres)
- **Управление зависимостями:** uv
- **Линт / формат:** ruff
- **Тесты:** pytest + pytest-django
- **Деплой:** Docker Compose + nginx + Let's Encrypt
- **Фронтенд:** серверный рендеринг Django Admin + кастомные шаблоны и CSS/JS поверх unfold (без React/SPA)

## Локальный запуск

### 1. Установка инструментов (один раз)

Python 3.12 — проверь: `python --version`

uv (менеджер пакетов):
```powershell
irm https://astral.sh/uv/install.ps1 | iex
```

Docker Desktop — опционально, для Postgres. На быстрый старт можно без него (SQLite).

### 2. Установка зависимостей

```powershell
uv sync
```

### 3. Окружение

```powershell
copy .env.example .env
```

### 4. База и админ

С Docker:
```powershell
docker compose up -d postgres
```

Без Docker — оставь `DATABASE_URL` пустым в `.env`, будет SQLite.

```powershell
uv run python manage.py migrate
uv run python manage.py createsuperuser
```

### 5. Запуск

```powershell
uv run python manage.py runserver
```

Открой http://127.0.0.1:8000/admin/

### Загрузить демо-данные

```powershell
uv run python manage.py import_students data/sample/students.csv
uv run python manage.py import_companies data/sample/companies.csv
```
